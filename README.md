# cctv_memory

CCTV 영상을 AI가 보고 **"누가 무슨 행동을 했는지"** 를 사건으로 기록해 두고,
**말로 검색**하는 관제 시스템.

> 예) "쓰러진 사람 찾아줘" → 그 장면·카메라·시각을 바로 표시
> 로컬 PC(RTX 4060 8GB)에서 동작. 화재는 열화상 카메라가 따로 담당.

<br>

## 1. 활용 데이터

**AI-Hub 지능형 관제 CCTV 영상 데이터**

- 6개 행동 유형: 쓰러짐 · 싸움 · 침입 · 군집 · 인파밀집 · 침수
- 각 영상에 **정답 라벨** 포함: 유형 + 사건이 일어난 정확한 시각 + 장면 설명
- 검증용 클립 19개 사용 (1920×1080, 1~3분)

> 라벨은 AI에게 안 보여줌.
> AI가 영상만 보고 맞히게 하고, 라벨은 **채점(정답지)** 으로만 사용.

<br>

## 2. 워크플로우

```
CCTV 영상
  ├─ ① 사람·차량 추적      누가 언제 들어와 얼마나 머물렀나
  ├─ ② 5초 구간으로 나눔
  ├─ ③ AI가 장면 설명       예: "두 사람이 몸싸움을 한다"
  ├─ ④ 비슷한 구간을 묶어 "사건"으로
  ├─ ⑤ 검색 DB에 저장
  │
  └─ [검색할 때]
       ⑥ 설명을 보고 유형 붙임 (싸움 / 침입 / …)
       ⑦ 화면에 표시 (검색 · 이력 · 알림)
```

역할 분담
- **사실**(체류·시각·입출입): 추적이 측정
- **행동의 의미**: AI가 설명
- **유형 분류**: 따로 작업 (언제든 교체)

<br>

## 3. 시도 변천

| 부분 | 이전 | 변경 | 이유 |
|---|---|---|---|
| 목적 | 화재 감지 | 사람 행동 기록 | 열화상카메라로 감지 |
| 저장 단위 | 5초 토막 | 사건 (토막 묶음) | 토막은 흩어져 못 읽음 |
| 체류시간 | AI가 추측 | 추적으로 측정 | AI는 시간 길이를 못 봄 |
| 유형 정하기 | VLM이 분류 | 임베딩 모델 분류 | VLM의 성능 최대화 |
| AI 설명 | 화재로 제한 | 사실 묘사 | 설명이 깨끗해야 분류 잘됨 |
| 유형 목록 | 고정 | 실제 데이터 기준 | 채점이 공정해짐 |
| 시각 | 영상 안 시간 | 실제 시각 | 관제는 실제 시각이 기본 |

**가장 큰 변화: "설명"과 "유형 분류"를 분리**

- 처음: AI가 영상 보며 설명 + 유형을 한 번에 정함
  → 새 유형(예: 침입) 추가하려면 **영상 전체 재분석** (수십 분)
- 지금: AI는 설명만, 유형은 나중에 "설명이 어느 유형과 의미가 비슷한가" 비교로 붙임
  → 유형 목록만 바꾸면 **재분석 없이 몇 초** 만에 다시 분류
- 비유: AI가 일기를 써두고 라벨은 따로 붙이는 것.
  라벨 기준이 바뀌면 일기는 그대로 두고 라벨만 다시 붙임

<br>

## 4. 결과

**완성**
- 영상 색인 → 말로 검색 → 카메라·시각별 이력 콘솔 동작

**검증됨**
- 유형 분류(설명 비교)가 샘플에서 정확
  - "몸싸움" → 싸움, "쓰러짐" → 낙상, "군집" vs "인파밀집"까지 구분
- 설명 프롬프트를 중립화하자 캡션 품질 개선
  - 전: 인파밀집을 *"액체가 쏟아져"* 로 오염
  - 후: *"사람들이 밀집해 길을 걷는다"* 로 정확히 묘사

**측정 예정** (재색인 후)
- 정확도 비교: AI 직접(A) vs 설명 비교(B) vs 정답(GT)
- 예상: A는 정해둔 목록에 없는 싸움·침입을 못 잡아 낮고, B는 잡아서 높음
